🕷 Session Replay Attack - Hardened

Paste a stolen session token below. The hardened app will show you exactly which security controls block the replay attempt and why.

Cross-app note (port 3001 vs 3002): The vulnerable and hardened apps use completely different JWT signing secrets. A token from port 3001 will fail verification on port 3002. If you appear logged in on both, those are independent sessions. Both apps use the same cookie name (session) on the same hostname, so logging into one may overwrite the other's cookie in your browser - use incognito windows to keep them separate.

To test cross-app replay: Copy a token from the vulnerable app's Cookie Vault, then paste it into the replay field below.

On the hardened app, manual injection still fails. Even if you successfully inject a stolen token into DevTools or curl, the server validates sessions against its server-side session store, checks IP and User-Agent binding, and enforces 15-minute token expiry. Defense-in-depth means no single bypass works.

Vulnerable vs Hardened - Side by Side

Vulnerable App (port 3001)

  • httpOnly: false - JS reads cookie
  • 7-day token expiry
  • No server-side session store
  • No device binding
  • sameSite: None
  • Logout only clears browser cookie
  • Stolen token works until expiry

Hardened App (port 3002)

  • httpOnly: true - JS gets nothing
  • 15-minute token expiry
  • Server-side session with revocation
  • IP + User-Agent binding
  • sameSite: Strict
  • Logout invalidates server session
  • Multiple layers block replay

Real-World Impact

Session cookie theft isn't theoretical - it's the #1 technique behind major breaches today.

Lumma Stealer Campaign (2025)

Lumma infostealer distributed via fake CAPTCHA pages and cracked software. Steals browser cookies, passwords, and crypto wallets. Microsoft disrupted its infrastructure in May 2025 after it infected 394,000+ Windows machines.

Snowflake Customer Breaches (2024)

Attackers used stolen session tokens (harvested by infostealers) to access Snowflake accounts of 165+ organizations including Ticketmaster (560M records) and AT&T (110M records). No MFA bypass needed - the cookies were enough.

Dutch Police Token Theft (2024)

A state actor stole session cookies from a Dutch police officer's machine via infostealer malware, accessing contact details of 63,000 officers. The attacker bypassed all authentication including the police network's MFA.

Cookie-Bite Attack on Azure Entra (2025)

Researchers demonstrated a browser extension that exfiltrates Azure Entra ID session cookies in real-time, bypassing MFA and Conditional Access Policies to access Microsoft 365 services including email, Teams, and SharePoint.

This lab demonstrates the exact techniques used in these attacks.
